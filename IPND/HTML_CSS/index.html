HTML Notes - Stages 0 and 1

HTML

  • HTML stands for Hypertext Markup Language.
  • It's a computer language that's interpreted by the web browser to display content to the users.
  • It's one of many document types that are stored on webservers
  • It's governed by a WorldWide standard.
  • HTML documents are linked together using hyperlinks

Tags

  • Tags tell the browser to display content in a certain way.
  • There is a start tag "< >" and a closing tag like so "</ >". For example the emphasis tag "<em> </em>"
  • Not all tags have a closing tag. For example the image element "<img>". These are known as void tags
  • Tags with the content in between are known as an element i.e. <tag> content <tag>

Inline vs Block

Inline elements work within a line of code. That is the surrounding content are all the same line and there are no line breaks.
A good example of this is the span tag "<span> </span>" and the emphasis tag "<em> </em>".

Block elements have an invisible block around them and starts on a new line. At the end of a block element the content that follow it start on a new line as well.
A good example of this is the div tag "<div> </div>" and the paragraph tag "<p> </p>".

HTML CSS & DOM

  • HTML5 - provides structure to the document
  • CSS (Cascading Style Sheets)- provides the styling behind the document
  • DOM (Document Object Model) - made of HTML elements in a tree-like structure and describes how the HTML document is accessed by the browser

Web Development Tools - Chrome Browser

To access the developer tools in Chrome, Click on the top-right hand corner → More Tools → Developer Tools. The developer tools show the DOM behind the webpage.

You can use it to manipulate both the CSS and HTML code to try out new code and see how it behaves without changing your underlying HTML and CSS code. You can just refresh the page if you don't like what you've done.

Introduction to Boxes

There are boxes around block level elements. These boxes can be manipulated by CSS, for example, displaying borders.

Linking to CSS stylesheet

It is recommended to link to the CSS file in your html file with <link rel="stylesheet" href="style.css">.
This is done in the <head> section of the HTML file.

CSS specifics and order

The order matters as to which CSS styling takes precendance:
Browser CSS (least) → Separate CSS file styling → In-html file styling → In Elements styling (most)

Specifics matter! If one selector is more specific than the others, then it takes precedence.

CSS format

CSS form: selector {declaration}
declaration is property: value;
selector can either be an element, a named class or a named ID. There's more to it but that's the basics of it.

Class vs ID for selectors

ID selector → ID's are unique identifiers that uniquely identifies an element in HTML. id is denoted by '#' eg #id
Class selector → refers to a group of elements that can be styled together. class is denoted by '.' eg .class

Box Model

  • content (inner) → padding → border → margin (outer)
  • "box-sizing: border-box;" declaration is the width of content + padding + border. More preferable.
  • "display: Flex;" declaration shows boxes(blocks) in-line as opposed to one below the other.
